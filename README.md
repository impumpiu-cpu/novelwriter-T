## Боль и решение: почему ИИ-продолжение всегда «немного не то»?

Инструменты для письма наперебой обещают «миллион слов одним кликом». Но если вы настоящий автор веб-романов или длинной прозы, вам знакомы эти **сводящие с ума моменты**:

- ❌ **Сеттинг рассыпается**: к двадцатой главе герой внезапно забывает свой козырь и даже путает имя злодея.
- ❌ **Все на одно лицо**: второстепенные персонажи у ИИ говорят одинаково, без души.
- ❌ **Чем дальше, тем хуже**: чтобы ИИ понимал прошлый сюжет, приходится каждый день «писать модели биографии героев».
- ❌ **Потеря контроля**: вы превращаетесь в «наборщика» при ИИ, который несётся вперёд, и непонятно, как вернуть его на задуманные рельсы.

**Главный барьер длинного романа — не скорость набора текста, а целостность мира и логика истории.**

**NovelWriter (NovWr)** создан именно для этого. Мы отказались от грубого подхода «запихнуть сотни тысяч слов в модель и надеяться на удачу». NovWr не «пишет роман за вас» — он даёт вам **всезнающего личного помощника (Copilot)**: точно помнит каждую связь между персонажами и каждый закон мира, позволяя вам сосредоточиться на самом главном — накале сюжета!

## 🚀 Главные козыри: не просто генерация, а «управление миром»

### 1. 🌍 Нерушимый движок архитектуры мира
Забудьте про бессмысленную склейку гигантских контекстов! Под капотом NovWr — **динамически растущая сеть сущностей и связей**.
При каждом продолжении система автоматически извлекает **именно те** фрагменты сеттинга, которые нужны **прямо сейчас** (персонажи в сцене, местные фракции, текущий уровень сил), и передаёт их ИИ. **Прощай, дрейф сеттинга — интеллект персонажей всегда на связи.**

### 2. ⚔️ Два клинка: Studio для творчества, Atlas для законов мира
Длинная проза — танец чувства и разума, поэтому у вас два элегантных пульта управления:
- **Studio (рабочее место писателя)**: максимально погружённый фронт письма. Мгновенные черновики в нескольких версиях, молниеносное сравнение на одном экране, ловля самых ярких искр вдохновения.
- **Atlas (карта мира)**: командный центр сеттинга с высоты птичьего полёта. Структурированное управление тысячами сущностей и связей, проверка конфликтов сеттинга, поиск «дыр в заделах» одним кликом.

### 3. 🛡️ Absolute Control: сдержанный страж, который только читает
Novel Copilot не лезет не в своё дело. Он следует предельно сдержанному когнитивному циклу «Find → Open → Read»:
как самый опытный редактор, он лишь по вашему запросу листает огромную библиотеку, обобщает и подаёт карточки предложений. **Принять или нет, что менять — стопроцентное право решения остаётся за вами.**

### 4. 🔒 BYOK и максимальная приватность: ваш труд неприкосновенен
Роман — кровь и плоть автора, здесь нет компромиссов по безопасности данных:
- **Полностью контейнеризованное самостоятельное развёртывание (Self-Host прежде всего)**: один Docker-запуск — и данные навсегда остаются на вашем диске или личном сервере.
- **Свободная замена движка**: OpenAI, Gemini, DeepSeek или ваша локальная дообученная модель… всё, что совместимо по интерфейсу, работает на вас!

## 🗺️ Типовой мастер-процесс

1. **Сотворение**: создайте новый мир в Library или потренируйтесь на встроенном примере «Путешествие на Запад».
2. **Фундамент**: с помощью **«Сгенерировать из сеттинга»** или **«Извлечь из глав»** быстро соберите команду главного героя и начальный мир.
3. **Развитие**: переключитесь в **Studio**, набросайте анонс следующей серии — и ИИ мгновенно выдаст несколько версий продолжения.
4. **Решение**: выберите лучшую основу, слегка подправьте и утвердите.
5. **Канонизация**: вернитесь в **Atlas** и занесите новые артефакты и главного злодея в сеть связей.
6. **Цикл**: кончилось вдохновение? Позовите **Copilot** за идеей прорыва.

## 🧠 Логика работы (архитектура)

### Общая схема

```text
                    ┌──────────────────────────────┐
                    │      React SPA (web/)        │
                    │  Library · Studio · Atlas ·  │
                    │        Novel Copilot         │
                    └──────────────┬───────────────┘
                                   │ REST /api (+NDJSON-стриминг)
                    ┌──────────────▼───────────────┐
                    │     FastAPI (app/main.py)    │
                    │  роутеры app/api/*: auth,    │
                    │  novels, world, copilot, llm │
                    └───────┬───────────┬──────────┘
                            │           │
              ┌─────────────▼──┐   ┌────▼─────────────────┐
              │  Ядро app/core │   │ SQLite / PostgreSQL  │
              │  генерация ·   │   │ (SQLAlchemy+Alembic) │
              │  модель мира · │   └────▲─────────────────┘
              │  индексация    │        │ очередь задач в БД
              └───────┬────────┘   ┌────┴─────────────────┐
                      │            │  Фоновый worker      │
              ┌───────▼────────┐   │ (workers/            │
              │ LLM-провайдер  │   │  background_jobs.py) │
              │ (OpenAI-совм.) │   └──────────────────────┘
              └────────────────┘
```

### Основные подсистемы

1. **Веб-сервер (`app/main.py`)** — точка входа FastAPI: проверка безопасности при старте (JWT-секрет, режим развёртывания), CORS, лимитер запросов, логирование каждого запроса с `X-Request-ID`, health-эндпоинты и раздача собранного SPA со SPA-fallback на `index.html`.

2. **Импорт рукописи (`app/core/ingest/`)** — загруженный `.txt` попадает в очередь; конвейер: приём → декодирование → распознавание структуры глав (`core/parser.py`, `core/chapter_headings.py` понимают китайскую/корейскую/латинскую нумерацию) → планирование фоновых задач → запись глав в БД. Язык текста определяется автоматически (`app/language_policy.py`: эвристики по письменностям CJK/латиницы).

3. **Модель мира (`app/core/world/`, `app/models.py`)** — сущности (персонажи, места, фракции…), связи между ними и «системы» (иерархии сил, хронологии, списки правил). Всё новое сначала становится **черновиком** и вступает в силу только после подтверждения автором (страница «Проверка черновиков» в Atlas). Источники черновиков:
   - **bootstrap** (`core/bootstrap*.py`) — автоматическое извлечение из глав: токенизация (jieba через Rust-модуль) → извлечение кандидатов → оконный анализ совместной встречаемости → уточнение через LLM;
   - **генерация из сеттинга** (`core/world/gen.py`) — автор вставляет текст описания мира, LLM структурирует его в черновики;
   - **импорт worldpack** (`core/world/worldpack_import*.py`) — загрузка готового файла мира с защитой подтверждённых записей от перезаписи.

4. **Индекс всей книги (`app/core/indexing/`, `rust/state_proto`)** — «window index» разбивает весь текст на перекрывающиеся окна и строит поисковую структуру (высокопроизводительная часть вынесена в Rust-расширение `_novwr_state_proto`). Индекс питает поиск Copilot по всей книге и помечается устаревшим при изменении глав.

5. **Конвейер продолжения (`app/core/generator.py`)** — сердце продукта:
   - собирается промпт: последние N глав контекста + **точечная инъекция сеттинга** через `LoreManager` (автомат Ахо–Корасик находит в контексте упоминания сущностей и подмешивает только релевантные записи мира);
   - `ai_client.py` вызывает OpenAI-совместимый API (стриминг NDJSON с фолбэком на обычный ответ, семафоры ограничения параллельных вызовов, учёт стоимости);
   - несколько версий генерируются параллельно; результат обрезается по границе предложения с учётом языка (`language_policy.trim_to_sentence_boundary`);
   - **постпроверка** (`continuation_postcheck.py`, `prose_check.py`): дрейф терминов относительно модели мира, повторяющиеся фразы, слишком длинные абзацы/предложения, «обобщающий тон» — всё подсвечивается в редакторе.

6. **Novel Copilot (`app/core/copilot/`)** — исследовательский агент только для чтения. Работает циклом инструментов «find (поиск по индексу) → open (раскрыть пакет свидетельств) → read (прочитать записи мира)», ведёт трассировку шагов (видна пользователю как «ход исследования») и выдаёт карточки предложений. Применение предложения (`apply.py`) — отдельный, явно подтверждаемый шаг с проверкой актуальности данных.

7. **Фоновый worker (`app/workers/background_jobs.py`)** — отдельный процесс (второй контейнер в docker-compose), опрашивает очередь задач в БД: импорт романов → отложенные сборки индекса → пересборки индекса → задачи bootstrap. «Аренда» (lease) с heartbeat защищает от двойного исполнения и зависших задач.

8. **Локализация** — интерфейс (`web/src/lib/uiMessages.ts` + пакеты `uiMessagePacks/*`) поддерживает **русский (по умолчанию)**, китайский и английский; язык переключается в Настройках. Бэкенд отдаёт сообщения Copilot на локали пользователя (ru/zh/en/ja/ko), а язык самого романа определяется независимо и влияет только на обработку текста и промпты.

### Путь запроса «сгенерировать продолжение»

```text
Studio → POST /api/novels/{id}/continuations (stream)
  → проверка квоты и конфигурации LLM (BYOK или переменные окружения)
  → generator.continue_novel_stream():
      контекст глав + инъекция LoreManager → системный промпт с целевой длиной
  → ai_client.generate_stream() → NDJSON-события в браузер (текст по мере генерации)
  → обрезка по границе предложения → постпроверка (дрейф/качество прозы)
  → результаты сохраняются; автор сравнивает версии и принимает одну
```

## 🎯 Кому это нужно?

- 👉 Тем, кто ведёт **длинный веб-роман / ранобэ / эпическое фэнтези / фанфик** и держит в руках огромную вселенную.
- 👉 Авторам, считающим, что «модель пишет неплохо, но всё забывает — приходится нянчиться каждый день».
- 👉 Настоящим писателям, убеждённым, что **ИИ — лишь инструмент, а душу ведёт автор**.
- 👉 Гикам, которые дорожат приватностью и не отдадут миллионы слов труда неизвестной SaaS-платформе.

Если вам нужна «фабрика воды», где по названию книги само пишется миллион слов, — закройте страницу.
Но если вы ищете абсолютное оружие, которое **убирает препятствия сеттинга, поддерживает поразительную целостность и выжимает максимум творческой эффективности**, — добро пожаловать в команду.

## 🚀 Быстрый старт (Quick Start)

### Установка в один клик (рекомендуется)

> Требуется установленный Docker; **Git не нужен**.

```bash
curl -fsSL https://raw.githubusercontent.com/Hurricane0698/novelwriter/main/install.sh | bash
```

Скрипт установки автоматически:

- установит `uv` (если ещё не установлен)
- установит CLI `novwr`
- инициализирует каталог по умолчанию `~/.novwr`
- скачает и запустит официальный Docker-образ

Основные команды:

```bash
novwr init
novwr run
novwr doctor
novwr upgrade
novwr uninstall
```

Каталог установки по умолчанию — `~/.novwr`. Если вы просто хотите пользоваться программой, **этого достаточно**.

### Альтернатива: Docker (для тех, кто знаком с Docker)

```bash
git clone https://github.com/Hurricane0698/novelwriter.git
cd novelwriter
cp .env.example .env
# отредактируйте .env и укажите конфигурацию вашего LLM API
docker compose up -d
```

После развёртывания откройте в браузере:

```text
http://localhost:8000
```

### Как выбрать

- **Большинству пользователей**: «установка в один клик» выше
- **Вы знакомы с Docker / хотите вручную управлять каталогами и compose**: «ручной Docker»
- **Вы собираетесь дорабатывать код**: см. «Локальная разработка» ниже

### Замечания по self-host

- **Режим работы**: по умолчанию запускается в режиме `selfhost`, фронтенд и бэкенд уже объединены. Из соображений безопасности по умолчанию слушается только `127.0.0.1:8000`.
- **Фоновые задачи**: Docker Compose запускает и веб-сервис, и фоновый worker; импорт загрузок, индексация всей книги и автоматическое извлечение мира выполняются worker'ом из очереди.
- **Инициализация**: при первом запуске система автоматически создаёт учётную запись администратора и встроенный демонстрационный проект «Путешествие на Запад».
- **Необходимое условие**: рабочий API-ключ большой языковой модели (совместимой со стандартом OpenAI).
- **Проверка совместимости**: на странице настроек есть «Проверить подключение» — проверка связности API, потокового ответа и поддержки JSON Mode.
- **Официальный образ**: `ghcr.io/hurricane0698/novelwriter:latest`

## 🛠️ Локальная разработка (Development)

Для доработки кода рекомендуется управлять Python-окружением через **uv**.

### Бэкенд (Backend)

```bash
scripts/setup_python_env.sh
cp .env.example .env
# отредактируйте .env
scripts/uv_run.sh uvicorn app.main:app --reload --port 8000
```

Единственный источник истины по Python-зависимостям — `pyproject.toml` + `uv.lock` + локальный `.venv`; версия Python по умолчанию — в [`.python-version`](.python-version).
`scripts/setup_python_env.sh` автоматически убеждается, что необходимое Rust-расширение `_novwr_state_proto` установлено; если в системе нет Rust toolchain, скрипт сразу завершится с подсказкой по исправлению, а не отложит проблему до импорта романа.

Запуск модульных тестов бэкенда:

```bash
scripts/uv_run.sh pytest tests/
```

### Фронтенд (Frontend)

```bash
cd web
npm ci
npm run dev
```

Dev-сервер фронтенда работает на `http://localhost:5173`.
В повседневной разработке и CI используйте `npm ci`: установка строго по `package-lock.json` исключает случайный дрейф на непроверенные версии зависимостей.

|---|---|---|
| `OPENAI_API_KEY` | **Да (при включённом ИИ)** | API-ключ LLM, используемый по умолчанию в selfhost |
| `OPENAI_BASE_URL` | Нет | Адрес OpenAI-совместимого шлюза |
| `OPENAI_MODEL` | Нет | ID модели по умолчанию |
| `MAX_CONTEXT_CHAPTERS` | Нет | Максимум последних глав, внедряемых при продолжении |
| `DEFAULT_CONTINUATION_TOKENS` | Нет | Лимит токенов продолжения по умолчанию |
| `JWT_SECRET_KEY` | Обязательна в production | Ключ подписи JWT |
| `DATABASE_URL` | Нет | Адрес подключения к БД; по умолчанию SQLite |
| `CORS_ALLOWED_ORIGINS` | Нет | Разрешённые источники в формате JSON-массива |

### Основные для hosted

| Переменная | Обязательна | Описание |
|---|---|---|
| `DEPLOY_MODE=hosted` | **Да** | Включает ветку поведения hosted |
| `HOSTED_INVITE_CODES` | Зависит от способа подключения | Список инвайт-кодов активации |
| `HOSTED_LLM_BASE_URL` | **Рекомендуется задать явно** | Адрес собственного LLM-шлюза hosted-платформы |
| `HOSTED_LLM_API_KEY` | **Рекомендуется задать явно** | Собственный LLM-ключ hosted-платформы |
| `HOSTED_LLM_MODEL` | **Рекомендуется задать явно** | ID собственной модели hosted-платформы |
| `HOSTED_GITHUB_LOGIN_ENABLED` | Нет | Показывать ли публично вход через GitHub OAuth |
| `HOSTED_MAX_USERS` | Нет | Жёсткий лимит активных пользователей hosted |
| `AI_MANUAL_DISABLE` | Нет | Ручное отключение всех функций ИИ |
| `AI_HARD_STOP_USD` | Нет | Жёсткая остановка hosted-ИИ по оценке затрат |
| `ENABLE_EVENT_TRACKING` | Нет | Включить ли продуктовую аналитику hosted |
| `ENABLE_DEBUG_ENDPOINTS` | Нет | Включить ли отладочные эндпоинты |

### Bootstrap / настройка затрат

| Переменная | Обязательна | Описание |
|---|---|---|
| `BOOTSTRAP_LLM_TIMEOUT_SECONDS` | Нет | Тайм-аут LLM-уточнения bootstrap |
| `BOOTSTRAP_MAX_CANDIDATES` | Нет | Лимит кандидатов bootstrap |
| `LLM_DEFAULT_INPUT_COST_PER_MILLION_USD` | Нет | Цена входных токенов для оценки затрат hosted |
| `LLM_DEFAULT_OUTPUT_COST_PER_MILLION_USD` | Нет | Цена выходных токенов для оценки затрат hosted |

> Ключевой момент hosted: фоновые задачи (например, bootstrap) в первую очередь читают `HOSTED_LLM_*`; не рассчитывайте, что hosted-worker «сам» подхватит только `OPENAI_*`.

Подробную конфигурацию см. в [`.env.example`](.env.example).
